Batch Block

Allows you to block a batch of documents in order to avoid its signature.

Request

Request URL: https://api-sbx.portaldeassinaturas.com.br/api/v2/document/batchBlock

Request headers
Content-Type (optional) string Media type of the body sent to the API.
Token string Subscription key which provides access to this API. Found in your Profile.

Request body

application/json

Sample


{
    "documents":[123456],
    "status": true 
}

Schema


{
    "type": "object",
    "required": [
        "documents",
        "status"
    ],
    "properties": {
        "documents": {
        "type": "array",
        "description": "List of documents IDs.",
        "items": {
            "type": "integer",
            "format": "int32"
        }
        },
        "status": {
        "type": "boolean"
        }
    },
    "example": "{\n  \"documents\":[123456], \n  \"status\": true\n}"
}

Responses

200 OK

Representations

application/json

Sample


{
"count": 0,
"documents": [{
    "id": 0,
    "blocked": true
    }]
}

{
    "type": "object",
    "required": [
        "count",
        "documents"
    ],

        "type": "integer",

                        "type": "integer",

                    },
                    "blocked": {
                        "type": "boolean"
                    }
                }
            }
        }
    }

Create

Sends a document that was previously uploaded by the upload API for the signature. You can send the document to Digital, Electronic or Server Signature using signers, electronicSigners and serverSigners attributes respectively. You can use all of them combined, also defining the order of signatures by the step attribute. The sender attribute is optional and should be used if you want says to the integration who is the user in your system that is sending the document. This information will be shown to the signer at the signature moment.

Request URL: https://api-sbx.portaldeassinaturas.com.br/api/v2/document/create
