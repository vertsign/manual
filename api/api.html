Sandbox

In order to offer the most flexible integration, we provide developers with a completely free and unlimited test environment. Check out the API Documentation that describes how to use the APIs.

It is important to test their features before starting the execution of the application in production environment to ensure that it will operate free from errors, in addition to making sure that it complies with all the rules defined by the use of APIs.

Also important is underline that Sandbox environment is completely independent and there is no link with Production. This means that you will have one API Token for Sandbox environment and another for Production environment. There is no native way to switch from test to live environment. Just use the appropriate Token and appropriate URL to perform a test or live integration.

Note

All signed documents on Sandbox are NOT legally signed. The certificate used as part of these signatures will NOT comply with legal expectations.

https://api-sbx.portaldeassinaturas.com.br/api/v2/

Authentication

The APIs authentication is carried out with the information of a Token you have to send in the header to authenticate all requests.

Token: It's a unique value generated for a product subscritpion.

Transport Protocol

For security reasons, all information passed through the APIs should be made through HTTPS protocol, which ensures a safe channel and eliminates the manually encrypted tokens. So, you can only use our API through HTTPS protocol and calls made over plain HTTP will fail.

Authentication Errors

Inexistent/wrong Token:The error 401 Unauthorized is returned.

Access denied due to invalid subscription key. Make sure to provide a valid key for an active subscription.

Error Codes

The API must return the correct HTTP codes for each request.

Código Erro Descrição
200 OK Success!!
400 Bad Request The request has invalid parameter(s).
401 Unauthorized The access token has not been informed or does not have access to the APIs.
404 Not Found The informed feature in the request has not been found.
406 Unauthorized Access not authorized to the feature.
413 Request is to Large The request is exceeding the limit allowed for your access token profile.
422 Too many Requests The consumer has exceeded the limit of requests by time.
500 Internal Server Error Unexpected error, something is broken in the API.
